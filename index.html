Features

Conqueror's Blade Bot Manager is the ultimate solution for House Management.

SQLite Database

Dynamically save and retrieve information within mere seconds.

Register System

Specific roles in your server can register members into the bot database.
Thanks to this you can keep track of every single member in your house easily.

User Bot Roles

This bot comes with 2 different assignable roles for your house to use.

  • Member
    Regular user of your house, they can see their own user information, see their party companions, set their units barracks and Main Weapon with Second Weapon information.
  • Captain
    Same as Member but this user have the rights for party management, these are usually the leaders of each house party.

Party System

Captains can create their own parties with a members limit, manage them (kick and invite members), Members can accept or decline party invitations. Roles with specific discord permissions can see a list of every party that has been created with it's Captain and Members information.

User Management

Roles with specific discord permissions (usually administrators) can set and modify registered users information such as set their weapons, roles and modify their units barracks.

Database Export

Roles with specific discord permissions (usually administrators) can export the database information to a .CSV file, this is useful for automation of Google Sheets files creation.

Admin Commands

  • ;register @TagName
    Registers the tagged user into the database.
  • ;registerall @TagNames
    Registers multiple tagged users into the database.
  • ;info @TagName
    Gets the registered tagged information through a DM.
  • ;role @TagName role
    Sets the registered tagged user role, possible options are Captain and Member
  • ;rename @TagName
    Sets the registered tagged user in-game name in the database to it's actual discord nickname.
  • ;setunits @TagName units
    Sets the registered tagged user units in the database. It can be a link to an image or unit names.
  • ;setweapons @TagName weapon1 (weapon2)
    Sets the registered tagged user weapons in the database. weapon1 is mandatory, weapon2 is optional.
  • ;dbexport
    Gets the database information through a DM, as an exported .CSV file ready to be uploaded to Google Sheets.

User Commands

  • ;info
    Gets it's own information through a DM (only if registered).
  • ;units units
    Sets it's own units as a text string (only if registered).
  • ;weapons weapon1 (weapon2)
    Sets it's own weapon information as a text string, weapon2 parameter is optional (only if registered).

Party Commands

Admin Party Commands

  • ;party list
    Gets all the house party's with their integrants and main weapon through a DM.

Captain Party Commands

  • ;party create
    Creates a party and assigns the user to it.
  • ;party invite @TagName
    Invites a registered user to join it's party, the user can ACCEPT or DECLINE on their DM with the Bot.
  • ;party kick @TagName
    Removes the party member from it's party, a Captain can only kick a member from it's own party.

Member Party Commands

  • ;party leave
    The member abandons their party.
